🔐 Configurer Google OAuth2

Guide étape par étape — Projet tp_twm (SFMC Bénin)

⚠️ Important : Ce guide vous permet d'obtenir les clés GOOGLE_CLIENT_ID et GOOGLE_CLIENT_SECRET à ajouter dans votre fichier .env. Ces clés permettent aux utilisateurs de se connecter avec leur compte Google.
1

Accéder à Google Cloud Console

Rendez-vous sur la Google Cloud Console :

https://console.cloud.google.com/

Connectez-vous avec votre compte Google (de préférence le compte Google de votre projet).

2

Créer un nouveau projet

  1. Cliquez sur le sélecteur de projet en haut (à côté du logo Google Cloud)
  2. Cliquez sur "Nouveau projet"
  3. Nom du projet : tp-twm-sfmc (ou votre choix)
  4. Cliquez sur "Créer"
  5. Attendez la création (quelques secondes), puis sélectionnez ce nouveau projet
3

Activer l'API Google+ (People API)

  1. Dans le menu latéral gauche, allez dans "APIs et services" → "Bibliothèque"
  2. Recherchez Google People API
  3. Cliquez sur le résultat et appuyez sur "Activer"

Cette API permet à NextAuth de récupérer le profil Google (nom, email, avatar) de l'utilisateur.

4

Configurer l'écran de consentement OAuth

  1. Allez dans "APIs et services" → "Écran de consentement OAuth"
  2. Choisissez le type d'utilisateur : "Externe" → Cliquez sur "Créer"
  3. Remplissez les champs obligatoires :
    • Nom de l'application : TransacGo / SFMC Bénin
    • E-mail d'assistance : votre adresse Gmail
    • Coordonnées du développeur : votre adresse Gmail
  4. Cliquez sur "Enregistrer et continuer" (ignorer les champs optionnels)
  5. Sur la page "Champs d'application" → Cliquez sur "Enregistrer et continuer"
  6. Sur la page "Utilisateurs de test" → Cliquez sur "Enregistrer et continuer"
  7. Résumé → Cliquez sur "Retour au tableau de bord"
5

Créer les identifiants OAuth 2.0

  1. Dans le menu, allez dans "APIs et services" → "Identifiants"
  2. Cliquez sur "+ Créer des identifiants" → "ID client OAuth"
  3. Type d'application : "Application Web"
  4. Nom : tp_twm_web_client
  5. Dans "URI de redirection autorisés", ajoutez ces 2 URLs :
    http://localhost:3000/api/auth/callback/google
    https://votre-domaine.com/api/auth/callback/google

    (La 2ème uniquement si vous déployez en production)

  6. Dans "Origines JavaScript autorisées", ajoutez :
    http://localhost:3000
  7. Cliquez sur "Créer"
6

Récupérer CLIENT_ID et CLIENT_SECRET

Une fenêtre popup s'affiche avec :

  • ID client OAuth 2.0 → c'est votre GOOGLE_CLIENT_ID
  • Secret client → c'est votre GOOGLE_CLIENT_SECRET

Copiez ces valeurs et mettez-les dans votre fichier .env :

GOOGLE_CLIENT_ID=123456789-abcdefghijklmnop.apps.googleusercontent.com GOOGLE_CLIENT_SECRET=GOCSPX-AbCdEfGhIjKlMnOpQrStUvWxYz

Important Ne jamais committer le fichier .env sur GitHub. Il est déjà dans .gitignore.

7

Relancer le serveur et tester

Après avoir mis à jour le .env, relancez le serveur de développement :

npm run dev

Accédez à la page de connexion :

http://localhost:3000/front/auth/login
  1. Cliquez sur "Se connecter avec Google"
  2. Choisissez votre compte Google
  3. Accordez les permissions
  4. Vous devez être redirigé vers /dashboard
  5. Vérifiez dans la base de données que l'utilisateur a été créé avec provider = 'google'

✅ Récapitulatif du fichier .env final

DATABASE_URL="postgresql://postgres:ro@127.0.0.1:5432/apiProjet" NEXTAUTH_URL="http://localhost:3000" NEXTAUTH_SECRET="sfmc-tp-twm-secret-key-2026-nextauth" GOOGLE_CLIENT_ID=votre_client_id_ici.apps.googleusercontent.com GOOGLE_CLIENT_SECRET=GOCSPX-votre_secret_ici
